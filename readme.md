# 🎨 ArtFolio: Modern Interactive Art Portfolio

## 📖 项目概述
这是一个基于现代前端技术栈构建的个人艺术作品集网页。风格参考典型的 Cargo 艺术网站，要求极强的视觉冲击力、现代感和交互性。
项目需要展示约 500 幅艺术画作（分为 5 个系列），以及艺术家介绍和展览信息。
**核心挑战与目标**：在保持复杂的动态背景和高质量图片展示的同时，彻底解决海量图片带来的加载慢、卡顿问题，实现极致流畅的用户体验。

## 🛠 技术栈要求 (AI 助手请严格遵守)
*   **框架**: Next.js (App Router) + React (利用服务端渲染和内置图片优化)
*   **样式**: Tailwind CSS (用于快速布局) + CSS Modules (用于复杂自定义样式)
*   **动画**: Framer Motion 或 GSAP (用于平滑的页面过渡和元素交互)
*   **特效**: React Three Fiber (R3F) 或原生 WebGL Shaders (用于生成背景的动态滤镜/低多边形效果，注意性能节流)
*   **性能组件**: 
    *   必须使用 Next.js 的 `next/image` 组件。
    *   画廊列表必须实现 **虚拟滚动 (Virtual Scrolling)** 或 **视口懒加载 (Intersection Observer)**。

## 📂 页面架构与视觉需求

### 1. 入口页 (Splash / Intro Page)
*   **视觉特征**: 极简文本 ("Please click me")，主视觉为位于屏幕中央的前景形象（通过可灵 AI 生成的免抠 PNG）。
*   **动态效果**: 背景是前景形象的某种变形、万花筒或动态滤镜效果。
*   **交互**: 鼠标悬浮前景图像时有轻微的浮动 (Hover) 效果，点击后平滑过渡至“主导航页”。

### 2. 主导航页 (Home / The Book Interface)
*   **视觉特征**: 背景或居中主体为一本“打开的书”或类似的平面承载物。
*   **交互导航**: 界面上散落着几个不同的独立形象（生物/物件），它们作为导航按钮使用。
    *   形象 A -> 指向【作者介绍】
    *   形象 B -> 指向【展览信息】
    *   形象 C/D/E -> 指向不同的【画作系列】
*   **实现建议**: 使用绝对定位 (`absolute`) 将按钮元素放置在底图特定位置。添加 Hover 时的放大或发光反馈。

### 3. 作品画廊页 (Gallery Pages - 共5个系列)
*   **视觉特征**: 画作在页面中心占据主体，下方/侧方配有中英文创作背景文字。
*   **交互**: **横向滚动 (Horizontal Scroll)**。支持鼠标滚轮左右滑动和点击左右箭头切换。
*   **背景特效**: 带有动态低多边形 (Low-poly) 或流体扭曲效果的背景，但背景颜色需能随着当前展示的画作主色调进行平滑渐变 (Color extraction)。
*   **🚨 性能强制要求**:
    *   由于有 500 张高清画作，绝不能一次性渲染所有的 DOM 节点。
    *   实现一个横向的 **Slider / Carousel**，确保 DOM 中最多只存在“当前画作”、“上一张”、“下一张” 3-5 个节点。
    *   图片必须懒加载 (Loading="lazy")，并且在进入视口前 100px 时才开始请求网络。
